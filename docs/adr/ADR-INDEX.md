## ADR Process

1. **Propose** — Create new ADR from template
2. **Review** — Hub owner reviews and discusses
3. **Accept** — ADR becomes binding doctrine
4. **Supersede** — New ADR replaces old one
5. **Deprecate** — ADR no longer applies

---

## Creating a New ADR

1. Copy `templates/adr/ADR.md` to `docs/adr/ADR-XXX-title.md`
2. Fill in all sections
3. Submit PR for review
4. Update this index when accepted

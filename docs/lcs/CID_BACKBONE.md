> **Created:** 2026-02-17

## What CID Is

CID (Cycle Identity) is the deterministic, versioned, append-only cycle anchor for the LCS execution engine. Every outreach cycle gets a CID. Every change to a cycle creates a new revision row. Historical rows are never modified.

CID lives under LCS. It is not a sub-hub. It is not stored in canonical identity. It is not signal logic. It is not strategy logic.

**CID = cycle anchor only.**

## CID Format

```
{sovereign_id}-{cycle_version}-R{revision}
```

Example: `acme-corp-001-202602-R1`

- Generated by `BEFORE INSERT` trigger — never manually typed
- No randomness, no UUID, no signal embedding
- Unique across the entire table

## Table: `cl.lcs_outreach_cycle`

| Column | Type | Description |
|--------|------|-------------|
| sovereign_id | TEXT | Logical FK to canonical company |
| outreach_id | TEXT | Logical FK to outreach lifecycle |
| cycle_version | VARCHAR(6) | Period identifier (YYYYMM) |
| revision | INT | Revision counter (starts at 1) |
| cid | VARCHAR | Deterministic CID (trigger-generated) |
| cycle_signal_code | CHAR(3) | 3-character signal type code |
| cycle_signal_version | INT | Signal schema version |
| cycle_state | cl.cycle_state | OPEN / EXECUTING / COMPLETED / FROZEN |
| signal_payload | JSONB | Optional context (not indexed) |
| created_at | TIMESTAMPTZ | Row creation time |
| frozen_at | TIMESTAMPTZ | Seal timestamp (NULL = mutable) |

**PK:** `(sovereign_id, outreach_id, cycle_version, revision)`
**Unique:** `(cid)`

## Revision Model

- First issuance: `revision = 1`
- Any change: INSERT new row with `revision + 1`
- No destructive updates — historical rows remain intact
- Frozen rows (`frozen_at IS NOT NULL`) reject all UPDATE attempts via trigger

## Immutability Enforcement

A `BEFORE UPDATE` trigger checks `OLD.frozen_at`:

```
IF OLD.frozen_at IS NOT NULL THEN
  RAISE EXCEPTION 'Frozen CID rows cannot be modified. Insert new revision instead.';
```

This is non-negotiable. There is no admin bypass.

## Current-Cycle View

```sql
SELECT * FROM cl.v_lcs_outreach_cycle_current;
```

Returns the latest revision per `(sovereign_id, outreach_id, cycle_version)` using `MAX(revision)`. LCS reads from this view. Never query the base table for current state.

## What CID Does NOT Store

- Signal logic or severity
- Priority or lifecycle domain
- Messaging performance or delivery metrics
- Message IDs (MID)
- Execution logs
- Strategy or evaluation intelligence

CID is identity-only. Everything else belongs elsewhere.
